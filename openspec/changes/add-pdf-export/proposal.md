# PDF 导出功能提案

## 概述

为 BiliNote 添加 PDF 导出功能，允许用户将生成的 Markdown 笔记导出为格式化的 PDF 文档。

## 背景

当前 BiliNote 可以生成 Markdown 格式的笔记，但用户无法直接导出为 PDF 格式进行分享或存档。虽然后端已安装 `markdown_pdf` 包，但尚未实现导出功能。

## 目标

1. 添加后端 API 端点，将 Markdown 转换为 PDF
2. 前端添加导出按钮，支持一键导出 PDF
3. 支持自定义 PDF 样式（可选）

## 影响范围

- **后端**: 新增 PDF 生成 API 端点
- **前端**: 添加导出按钮和下载功能
- **依赖**: 使用现有的 `markdown_pdf` 包

## 实现方案

### 方案选择

**方案 A: 后端生成 PDF（推荐）**
- 优点：统一处理逻辑，支持服务器端缓存，前端轻量
- 缺点：增加服务器负载
- 实现：后端使用 `markdown_pdf` 将 Markdown 转为 PDF，前端通过 API 下载

**方案 B: 前端生成 PDF**
- 优点：减轻服务器负载
- 缺点：需要额外的前端库，处理中文可能有问题
- 实现：使用 `jsPDF` + `markdown-it` 前端渲染

**推荐方案 A**，理由是后端已有 `markdown_pdf` 依赖，且能更好地处理中文和复杂样式。

### 技术细节

1. **后端 API**
   - 新增 `GET /api/export/pdf/{task_id}` 端点
   - 读取缓存的 Markdown 文件
   - 使用 `markdown_pdf` 转换为 PDF
   - 返回 PDF 文件流

2. **前端实现**
   - 在笔记详情页添加"导出 PDF"按钮
   - 调用 API 触发下载
   - 显示加载状态和成功提示

3. **样式配置**
   - 使用 CSS 定义 PDF 样式
   - 支持中文字体
   - 保持与 Markdown 预览一致的视觉效果

## 风险与限制

1. **中文支持**: `markdown_pdf` 需要配置中文字体
2. **图片处理**: Markdown 中的图片需要能正确访问
3. **性能**: 大文档转换可能较慢，需要超时处理

## 依赖

- 现有: `markdown_pdf==1.7`
- 新增: 无（使用现有依赖）
