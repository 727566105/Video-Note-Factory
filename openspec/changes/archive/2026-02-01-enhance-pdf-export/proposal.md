# PDF 导出增强功能提案

## 概述

在基础 PDF 导出功能基础上，添加缓存机制、自定义样式、批量导出和导出历史记录等增强功能，提升用户体验和系统性能。

## 背景

基础 PDF 导出功能（Phase 1）已实现，但存在以下改进空间：
- 每次导出都需要重新生成 PDF，性能较低
- 用户无法选择不同的 PDF 样式
- 无法批量导出多个笔记
- 没有导出历史记录，无法追溯或重新下载

## 目标

1. 添加 PDF 缓存机制，显著提升重复导出的响应速度
2. 提供多种 PDF 样式主题，满足不同使用场景
3. 支持批量导出多个笔记为 ZIP 文件
4. 记录导出历史，支持查询和重新下载

## 影响范围

- **后端**: 扩展 `/api/export` 路由，新增 5 个 API 端点
- **前端**: 可选的样式选择器和批量导出 UI（后续实现）

## 实现方案

### T2.1 PDF 缓存机制

**策略**: 基于文件修改时间的智能缓存

- 缓存文件命名: `{task_id}_{style}.pdf`
- 缓存验证: 比较 PDF 与 Markdown 的修改时间
- 缓存命中: 直接返回缓存的 PDF 文件
- 缓存失效: Markdown 修改后自动重新生成

**性能提升**:
- 首次导出: ~0.8 秒
- 缓存命中: ~0.02 秒 (约 45 倍提升)

### T2.2 自定义样式选项

**样式主题**:

| 样式 ID | 名称 | 适用场景 |
|---------|------|----------|
| default | 默认样式 | 屏幕阅读，色彩丰富 |
| simple | 简洁样式 | 极简设计，清爽干净 |
| print | 打印样式 | 打印输出，黑白配色 |
| academic | 学术样式 | 学术论文，首行缩进 |

**API 设计**:
- `GET /api/export/styles` - 获取可用样式列表
- `GET /api/export/pdf/{task_id}?style=simple` - 指定样式导出

### T2.3 批量导出功能

**功能设计**:
- 接收任务 ID 列表，限制最多 50 个
- 支持指定统一样式
- 打包为 ZIP 文件下载
- 返回成功/失败统计

**API 设计**:
- `POST /api/export/batch` - 批量导出
- 请求体: `{"task_ids": ["id1", "id2", ...]}`
- 响应头: `X-Generated-Count`, `X-Failed-Count`

### T2.4 导出历史记录

**数据存储**: JSON 文件 (`.export_history.json`)

**记录内容**:
- 任务 ID
- 样式主题
- 笔记标题
- 导出时间戳
- PDF 缓存文件名

**API 设计**:
- `GET /api/export/history?limit=50` - 获取导出历史
- `GET /api/export/history/{task_id}` - 获取任务历史
- `GET /api/export/redownload/{task_id}` - 重新下载 PDF

## 依赖

- 现有: Phase 1 PDF 导出功能
- 新增: 无（使用 Python 标准库）

## 风险与限制

1. **缓存存储**: 每个任务每个样式会产生一个缓存文件，占用磁盘空间
2. **历史记录**: JSON 文件存储不适合大规模场景，后续可迁移到数据库
3. **批量导出**: 大量笔记可能导致内存占用过高
