# 导出图文功能 - 技术设计

## 架构概述

```
用户点击"导出图文"按钮
    ↓
前端：调用图文导出 API
    ↓
后端：接收请求，读取笔记内容
    ↓
后端：根据模板渲染 HTML
    ↓
后端：使用 html2canvas 转换为图片
    ↓
后端：返回图片文件流
    ↓
前端：触发图片下载
```

## 技术选型

### 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **html2python** | 纯 Python，无需浏览器 | 渲染质量有限，字体支持弱 | ⭐⭐ |
| **Playwright** | 渲染质量高，支持现代 CSS | 需要安装浏览器，资源占用高 | ⭐⭐⭐⭐ |
| **weasyprint** | PDF 转图片方便 | 主要用于 PDF，图片功能弱 | ⭐⭐ |
| **前端直接使用 html2canvas** | 减轻后端压力，用户可见 | 需要处理跨域、字体等问题 | ⭐⭐⭐ |

### 推荐方案：后端 Playwright + 前端 html2canvas

**理由：**
1. 后端 Playwright 确保渲染一致性和质量
2. 前端 html2canvas 提供实时预览和客户端备选方案
3. 两者互补，提高系统可靠性

### 渲染流程

```python
async def export_note_as_image(
    task_id: str,
    template: str = "xiaohongshu",
    width: int = 1080,
    format: str = "png"
) -> Response:
    """
    图文导出流程：
    1. 读取笔记 Markdown 内容
    2. 根据 template 渲染 HTML
    3. 使用 Playwright 渲染为图片
    4. 返回图片文件流
    """
    # 1. 读取笔记
    markdown_content = load_markdown(task_id)

    # 2. 渲染 HTML
    html_content = render_template(markdown_content, template)

    # 3. 转换为图片
    image_bytes = await html_to_image(html_content, width, format)

    # 4. 返回
    return Response(content=image_bytes, media_type=f"image/{format}")
```

## 性能优化

1. **缓存机制**：生成的图片缓存到本地，相同请求直接返回
2. **异步处理**：使用后台任务处理图片生成
3. **队列管理**：避免并发过多导致服务器压力
4. **图片压缩**：自动压缩图片，减少传输大小

## 错误处理

```python
try:
    image_bytes = await generate_image(html_content)
except RenderingError as e:
    logger.error(f"图片渲染失败: {e}")
    raise HTTPException(
        status_code=500,
        detail="图文生成失败，请稍后重试"
    )
except TimeoutError:
    raise HTTPException(
        status_code=504,
        detail="图文生成超时，请减少笔记内容或联系管理员"
    )
```

## 安全考虑

1. **输入验证**：限制图片宽度（最大 1920px），防止资源滥用
2. **速率限制**：每个用户每分钟最多 10 次图文导出
3. **内容过滤**：敏感内容不生成图文
4. **文件清理**：定期清理过期的缓存图片
