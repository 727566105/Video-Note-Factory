# image-export Specification

## Purpose

提供将生成的笔记内容渲染为图片格式的功能，方便用户在社交媒体平台分享。图片样式参考小红书分享卡片的设计风格。

## ADDED Requirements

### Requirement: 图文导出 API 端点

系统 SHALL 提供 `/api/export/image/{task_id}` API 端点，该端点 MUST 支持将笔记内容渲染为指定模板样式的图片，并返回图片文件流。

#### Scenario: 成功导出小红书风格图片

**Given** 一个有效的任务 ID（笔记已成功生成）
**And** 请求参数 template 为 "xiaohongshu"
**When** 客户端调用 `GET /api/export/image/{task_id}?template=xiaohongshu`
**Then** 系统返回 PNG 图片文件流
**And** Content-Type 为 `image/png`
**And** 图片使用小红书风格样式（温暖配色、卡片布局）
**And** 文件名为 `note_{title}_{template}.png`

#### Scenario: 使用默认模板导出

**Given** 一个有效的任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}`（未指定 template 参数）
**Then** 系统使用默认模板（xiaohongshu）
**And** 返回对应模板样式的图片

#### Scenario: 笔记不存在

**Given** 一个无效或不存在任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}`
**Then** 系统返回 404 状态码
**And** 返回错误信息 "笔记不存在，请确认任务 ID 正确"

#### Scenario: 笔记内容为空

**Given** 一个有效的任务 ID
**And** 对应的笔记内容为空
**When** 客户端调用 `GET /api/export/image/{task_id}`
**Then** 系统返回 400 状态码
**And** 返回错误信息 "笔记内容为空，无法生成图文"

#### Scenario: 不支持的模板

**Given** 一个有效的任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}?template=invalid_template`
**Then** 系统返回 400 状态码
**And** 返回错误信息 "不支持的模板类型"

### Requirement: 图片模板样式系统

系统 SHALL 提供多种预设的图片模板样式，每个模板 MUST 包含独立的配色方案、布局设计和字体设置。

#### Scenario: 小红书风格模板

**Given** 用户选择 xiaohongshu 模板
**When** 系统渲染图片
**Then** 图片使用温暖米色背景（#FFF5E6）
**And** 使用红色作为主色调（#FF2442）
**And** 采用卡片式布局，带圆角和阴影
**And** 标题使用加粗的大字号
**And** 内容区与元数据区有清晰的分割线
**And** 底部包含"Generated by Video Note Factory"水印

#### Scenario: 简约风格模板

**Given** 用户选择 simple 模板
**When** 系统渲染图片
**Then** 图片使用白色背景（#FFFFFF）
**And** 使用灰色作为文字颜色（#2c3e50）
**And** 采用极简布局，无边框和装饰
**And** 字体使用 Noto Sans SC
**And** 间距适中，排版清爽

#### Scenario: 学术风格模板

**Given** 用户选择 academic 模板
**When** 系统渲染图片
**Then** 图片使用浅灰色背景（#F5F5F5）
**And** 使用衬线字体（Noto Serif SC）
**And** 首行缩进（2em）
**And** 标题居中对齐
**And** 使用正式的排版风格

---

### Requirement: 图片尺寸自定义

系统 SHALL 支持自定义导出图片的宽度，MUST 支持常用分辨率并限制最大宽度以防止资源滥用。

#### Scenario: 使用自定义宽度

**Given** 一个有效的任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}?width=1920`
**Then** 导出图片的宽度为 1920px
**And** 高度根据内容自动计算，保持长宽比合理

#### Scenario: 使用默认宽度

**Given** 一个有效的任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}`（未指定 width）
**Then** 导出图片使用默认宽度 1080px

#### Scenario: 宽度超限

**Given** 一个有效的任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}?width=4000`
**Then** 系统返回 400 状态码
**And** 返回错误信息 "图片宽度超出限制，最大支持 1920px"

#### Scenario: 宽度无效

**Given** 一个有效的任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}?width=abc`
**Then** 系统返回 400 状态码
**And** 返回错误信息 "图片宽度必须是数字"

---

### Requirement: 图片格式支持

系统 SHALL 支持 PNG 和 JPG 两种图片格式导出，PNG MUST 为默认格式。

#### Scenario: 导出 PNG 格式

**Given** 一个有效的任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}?format=png`
**Then** 系统返回 PNG 格式图片
**And** Content-Type 为 `image/png`
**And** 图片支持透明背景（模板需要时）

#### Scenario: 导出 JPG 格式

**Given** 一个有效的任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}?format=jpg`
**Then** 系统返回 JPG 格式图片
**And** Content-Type 为 `image/jpeg`
**And** 图片质量不低于 85%

#### Scenario: 使用默认格式

**Given** 一个有效的任务 ID
**When** 客户端调用 `GET /api/export/image/{task_id}`（未指定 format）
**Then** 系统默认导出 PNG 格式

---

### Requirement: 图片内容渲染

系统 SHALL 正确渲染笔记的 Markdown 内容，包括标题、段落、列表、代码块、引用块和图片。

#### Scenario: 渲染标题层级

**Given** Markdown 内容包含多级标题（H1-H4）
**When** 系统渲染图片
**Then** H1 标题显示为大号加粗字体
**And** H2-H4 标题字号依次递减
**And** 标题层级关系清晰
**And** 标题与正文之间有适当间距

#### Scenario: 渲染列表

**Given** Markdown 内容包含有序列表和无序列表
**When** 系统渲染图片
**Then** 列表项正确缩进
**And** 列表标记显示清晰（数字或圆点）
**And** 列表项之间有适当间距

#### Scenario: 渲染代码块

**Given** Markdown 内容包含代码块
**When** 系统渲染图片
**Then** 代码块使用等宽字体
**And** 代码块有背景色区分
**And** 长代码块支持自动换行或滚动

#### Scenario: 渲染引用块

**Given** Markdown 内容包含引用块
**When** 系统渲染图片
**Then** 引用块有特殊的样式（如左侧边框、斜体）
**And** 引用块与正文有清晰区分

#### Scenario: 渲染内嵌图片

**Given** Markdown 内容包含图片引用（`![alt](path)`）
**And** 图片文件存在于 `/static/` 目录
**When** 系统渲染图片
**Then** 内嵌图片正确显示
**And** 图片大小适配容器
**And** 图片位置正确（不溢出边界）

---

### Requirement: 图片缓存机制

系统 SHALL 实现图片缓存机制，相同参数的导出请求 MUST 直接返回缓存图片以提高响应速度。

#### Scenario: 缓存命中

**Given** 之前已成功导出某笔记的 xiaohongshu 模板图片
**And** Markdown 文件未发生变化
**When** 客户端再次调用 `GET /api/export/image/{task_id}?template=xiaohongshu`
**Then** 系统直接返回缓存的图片文件
**And** 响应时间小于 1 秒
**And** 不重新执行渲染操作

#### Scenario: 缓存失效

**Given** 已存在某笔记的缓存图片
**And** Markdown 文件已被修改（修改时间更新）
**When** 客户端调用 `GET /api/export/image/{task_id}`
**Then** 系统检测到缓存失效
**And** 重新生成图片
**And** 更新缓存文件

#### Scenario: 缓存文件命名

**Given** 一个任务 ID 为 "abc123"
**And** 选择的模板为 "xiaohongshu"
**When** 系统生成或更新缓存图片
**Then** 缓存文件名为 `abc123_image_xiaohongshu.png`
**And** 文件保存在 `note_results/` 目录

---

### Requirement: 性能要求

图文导出操作 SHALL 在性能约束内完成：普通笔记（5000字以下）MUST 在 10秒内完成，长笔记（10000字以下）SHALL 在 30秒内完成。

#### Scenario: 普通笔记导出

**Given** 一个包含 5000 字的 Markdown 笔记
**When** 执行图文导出操作
**Then** 导出时间少于 10 秒

#### Scenario: 长笔记导出

**Given** 一个包含 10000 字的 Markdown 笔记
**When** 执行图文导出操作
**Then** 导出时间少于 30 秒

#### Scenario: 超时处理

**Given** 导出操作超过 60 秒
**When** 仍未完成
**Then** 系统返回 504 状态码
**And** 返回错误信息 "图文生成超时，请减少笔记内容或联系管理员"

#### Scenario: 并发限制

**Given** 系统正在处理多个图文导出请求
**When** 并发请求数量超过 5 个
**Then** 系统对额外请求返回 429 状态码
**And** 返回错误信息 "服务繁忙，请稍后重试"

---

### Requirement: 前端导出按钮

笔记详情页工具栏 SHALL 提供"导出图文"按钮，该按钮 MUST 在笔记生成成功后可用，点击后 SHALL 显示模板选择对话框。

#### Scenario: 显示导出图文按钮

**Given** 用户在笔记详情页
**And** 笔记已成功生成（status 为 SUCCESS）
**When** 页面渲染
**Then** 显示"导出图文"按钮
**And** 按钮位置与"导出 PDF"按钮并列
**And** 按钮图标表示图片或相册

#### Scenario: 导出图文按钮禁用状态

**Given** 用户在笔记详情页
**And** 笔记尚未生成（status 不是 SUCCESS）
**When** 页面渲染
**Then** "导出图文"按钮处于禁用状态
**And** 按钮显示为灰色

#### Scenario: 点击导出图文按钮

**Given** "导出图文"按钮可用
**When** 用户点击按钮
**Then** 显示模板选择对话框
**And** 对话框列出所有可用模板
**And** 每个模板显示预览图和名称

---

### Requirement: 模板选择对话框

模板选择对话框 SHALL 展示所有可用模板，包括预览图、模板名称和描述，用户 MUST 选择模板后才开始导出。

#### Scenario: 显示模板列表

**Given** 模板选择对话框打开
**When** 对话框渲染完成
**Then** 显示至少 3 种模板选项
**And** 每个选项包含：
  - 模板预览图
  - 模板名称（如"小红书风格"）
  - 模板描述（如"温暖配色，卡片式布局，适合社交分享"）
**And** 模板按推荐顺序排列

#### Scenario: 选择模板并导出

**Given** 模板选择对话框打开
**When** 用户点击某个模板选项
**And** 点击确认按钮
**Then** 对话框关闭
**And** 开始导出流程
**And** 显示加载状态

#### Scenario: 取消模板选择

**Given** 模板选择对话框打开
**When** 用户点击取消按钮或对话框外部区域
**Then** 对话框关闭
**And** 不执行导出操作
**And** 页面状态保持不变

---

### Requirement: 导出进度反馈

系统 SHALL 在导出过程中提供清晰的进度反馈，包括加载状态和成功/失败提示。

#### Scenario: 显示导出中状态

**Given** 用户选择了模板并确认导出
**When** 导出请求发送后
**Then** "导出图文"按钮显示加载动画
**And** 按钮文字变为"生成中..."
**And** 按钮处于禁用状态

#### Scenario: 导出成功

**Given** 图文导出请求成功完成
**When** 图片返回给客户端
**Then** 停止加载动画
**And** 按钮恢复可用状态
**And** 显示成功提示 "图文导出成功"
**And** 自动触发图片下载

#### Scenario: 导出失败

**Given** 图文导出请求失败
**When** 接收到错误响应
**Then** 停止加载动画
**And** 按钮恢复可用状态
**And** 显示错误提示 "图文导出失败：{错误信息}"
**And** 错误信息友好且可操作（如建议减少笔记内容）

---

### Requirement: 图片下载处理

系统 SHALL 正确处理生成的图片文件下载，包括文件名编码和浏览器兼容性。

#### Scenario: 标准文件名

**Given** 导出成功
**And** 笔记标题为"如何学习 Python"
**And** 选择的模板为"xiaohongshu"
**When** 触发浏览器下载
**Then** 文件名为 `如何学习Python_xiaohongshu.png`
**And** 文件名已进行安全处理（移除特殊字符）

#### Scenario: 文件名包含特殊字符

**Given** 导出成功
**And** 笔记标题包含特殊字符（如 `视频：2024/12/31 测试`）
**When** 触发浏览器下载
**Then** 文件名中的特殊字符被移除或替换
**And** 文件名仍然是有效的文件系统名称

#### Scenario: 文件名过长

**Given** 导出成功
**And** 笔记标题超过 100 个字符
**When** 触发浏览器下载
**Then** 文件名被截断到 100 个字符
**And** 保留文件扩展名 (.png)

#### Scenario: 跨浏览器兼容

**Given** 用户使用不同的浏览器（Chrome、Firefox、Safari）
**When** 下载图片
**Then** 所有浏览器都能正确触发下载
**And** 文件名编码正确处理（UTF-8）
